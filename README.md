## 项目介绍

FleetSim 名字由来解释：

- **Fleet**：车队，表示管理的车辆队伍  
- **Sim**：Simulation（模拟），表示模拟业务场景  

项目适合作为教学示例，也可在此基础上扩展成为完整的车辆运输调度系统。

功能特性：

- 基于 **Gin** 的 RESTful API
- 模拟车辆状态（行驶、停车、空闲）
- 动态路径规划与调度
- 支持多车队、多车辆管理
- 实时查询车辆位置与状态
- 可视化前端（可选，基于 Vue 或其他前端框架）
- 支持导入导出车辆数据和路径配置

## 技术选型

**开发语言**：Go

**后端框架**：[Gin](https://github.com/gin-gonic/gin)

**ORM**：[GORM](https://gorm.io/)

**数据库**：MySQL

**接口文档**：[Swagger](https://github.com/swaggo/swag)

**依赖管理**：Go Modules

**部署**：Docker / Kubernetes

## 项目结构

```
fleetsim/
├─ cmd/                   				# 可执行入口
│   └─ main.go
├─ config/                				# 配置文件
│   └─ config.go
├─ internal/
│   ├─ api/               				# HTTP接口层（Controller）
│   │   └─ vehicle.go
│   ├─ service/           				# 业务逻辑层
│   │   └─ vehicle_service.go
│   ├─ repository/        				# 数据库操作层（Mapper/Repository）
│   │   └─ vehicle_repo.go
│   ├─ model/             				# 数据库实体
│   │   └─ vehicle.go
│   └─ common/            				# 公共工具，如统一返回Result
│       └─ response.go
├─ scripts/               				# 数据库初始化或迁移脚本
├─ docs/                  				# Swagger 自动生成文档与代码样例文档
├─ go.mod
└─ go.sum
```

## 构建方式

### 1. 基础环境

**操作系统**：macOS / Linux / Windows

**Go**：1.21+（需要在 `PATH` 中可用）

**MySQL**：5.7+ 或 8.x以上

**Git**：用于拉取代码

> 项目较新，初次上传于2025年11月13日，mysql与go的版本使用最新的LTS版本目前实测不会出现任何问题。

### 2. 克隆仓库

- 选择克隆方式

```bash
// SSH连接
git clone git@github.com:kiritosuki/fleetsim.git
// Http连接
git clone https://github.com/kiritosuki/fleetsim.git
```

- 进入项目根目录`fleetsim`

### 3. 配置数据库

- 创建名为`routing`的database
- 执行`/scripts`下的数据库初始化脚本`routing.sql`
- 修改`/config/config.go`中的数据库配置信息（详情见代码注释）

### 4. 安装依赖

```go
go mod tidy
```

### 5. 启动服务

```go
go run ./cmd/main.go
```

或者从`goland` / `vscode`启动项目

## 贡献指南

1. Fork 本仓库，再克隆 Fork 后的仓库到本地
2. 添加本仓库为上游

```bash

git remote add upstream git@github.com:kiritosuki/fleetsim.git
// 将源仓库的最新内容fetch到本地
git fetch upstream
```

3. 新建分支并修改

```bash
git branch feature/xxx
git switch feature/xxx
# 修改/增添代码
git add .
git commit -m "详细描述"
git push -u origin feature/xxx
```

4. 在 github 打开你fork的仓库，填写标题和说明，提交PR

> 请遵守golang与gin社区开发规范，在申请PR前请确保所有代码均通过单元测试
